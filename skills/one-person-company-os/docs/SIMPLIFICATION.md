## 现在：只要 1 条命令

### ✅ 开始一个新项目

```bash
# 只要这一条！
node opc start "做一个博客系统"
```

**系统自动做的事（你完全不用管）：**
1. ✅ 自动激活 CEO 角色
2. ✅ 问你 3 个关键问题（目标用户、核心价值、时间）
3. ✅ 自动激活 PM 角色
4. ✅ 自动生成 PRD
5. ✅ 自动创建 PRD 交付物
6. ✅ 自动 CEO 签收
7. ✅ 自动激活 Architect 角色
8. ✅ 自动生成架构设计
9. ✅ 自动创建架构交付物
10. ✅ 自动 CEO 签收
11. ✅ 保存所有文档到 `.opc/memory/shared/`

**你只需要做的事：**
- 回答 3 个问题
- 然后直接去写代码！

```bash
node agents/deliverable-manager.js list
```

## 核心设计原则：隐形管控

> 好的流程管控应该像空气——无处不在，但你感觉不到它的存在。

| 机制 | 用户感知 | 实际效果 |
|------|----------|----------|
| 角色依赖检查 | 无感知 | 缺少工具时才提醒 |
| 独立记忆系统 | 无感知 | 自动积累经验，越来越聪明 |
| 交付物签收 | 无感知 | 自动创建、自动签收、自动归档 |
| 质量门禁 | 快卡住时才提醒 | 不会让你在错误的道路上走太远 |

**因为信任所以简单**——系统在后台默默地帮你把控质量、积累经验、维护流程，但你完全感觉不到它的存在。你只需要专注于写代码和做决策。

## 后续简化方向（持续降低使用成本）

### V2.0：连命令都不用敲

```
# 直接在代码根目录运行
opc
```

自动检测项目状态，自动推荐下一步。

### V3.0：和 Git 集成

```
git commit -m "feat: xxx"
```

自动触发 QA 检查、安全扫描、更新交付物状态。

### V4.0：完全集成到 IDE

打开 VS Code 时，自动显示：
- 当前项目阶段
- 下一步该做什么
- 质量分和改进建议
- 历史经验提醒
