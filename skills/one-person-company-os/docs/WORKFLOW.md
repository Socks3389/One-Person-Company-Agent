# OPC OS 完整工作流

## 七阶段流程

| 阶段 | 角色 | 输入 | 输出 | 出口条件 |
|------|------|------|------|----------|
| 0. 创意发起 | CEO | 用户想法 | 商业简报 | ✅ 目标、范围、时间三要素清晰 |
| 1. 产品定义 | PM | 商业简报 | PRD + 验收标准 | ✅ MVP 边界清晰，验收可量化 |
| 2. 架构设计 | Architect | PRD | 架构设计 + API 契约 | ✅ 技术选型有理由，风险有预案 |
| 3. 代码实现 | Engineers | 架构设计 | 代码 + 实现文档 | ✅ 测试覆盖，错误处理完整 |
| 4. 质量验证 | QA | 实现代码 | 测试报告 | ✅ 所有验收条件通过 |
| 5. 安全审计 | Security | 代码 + 测试报告 | 安全报告 | ✅ 无高危漏洞，Secret 无泄露 |
| 6. 发布上线 | Release Manager | 所有交付物 | 发布记录 | ✅ 备份、回滚、监控、冒烟都到位 |
| 7. 复盘优化 | 全员 | 整个过程 | 记忆更新 | ✅ 每个角色都更新了经验库 |

## RACI 矩阵

| 阶段 | CEO | PM | Architect | Web-FE | API-BE | QA | Security | RM |
|------|-----|----|-----------|--------|--------|----|----------|-----|
| 创意发起 | R | | | | | | | |
| 产品定义 | A | R | C | | | | | |
| 架构设计 | A | C | R | | | | | |
| 代码实现 | | | C | R | R | | | |
| 质量验证 | A | | | | | R | C | |
| 安全审计 | A | | | | | | R | |
| 发布上线 | R | | | | | A | A | A |
| 复盘优化 | R | I | I | I | I | I | I | I |

R=执行, A=审批, C=咨询, I=通知

## 质量门禁（< 80 分卡壳）

| 检查项 | 扣分 |
|--------|------|
| 验收条件不可量化 | -10 |
| MVP 边界不清晰 | -15 |
| 技术选型没有理由 | -20 |
| 没有错误处理 | -15 |
| 没有单元测试 | -20 |
| 没有回滚预案 | -30 |
| Secret 泄露 | BLOCKER |
